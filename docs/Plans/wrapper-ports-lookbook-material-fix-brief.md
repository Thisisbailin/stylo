# Mission Brief — 包装器端口、Lookbook 材质与收起稳定性修复

## 目标

- Lookbook 与 Manus 的包装成员连接线在任意相对位置都连续、自然，不产生回环或断裂感。
- Lookbook 完全脱离原子节点 Card 外壳，呈现为略微张开的精装摄影集。
- 移除页芯中过密的横向条纹，仅保留少量真实纸张层次。
- Lookbook 封面和内页不使用聚光、高光、发光或内容媒体投影；立体感来自几何层叠与书体外部接触阴影。
- 单击收起/展开一次只切换一次，不出现收起后立即恢复。
- 新建及遗留默认 Markdown 文本节点在界面中统一显示为“文本”。

## 不做

- 不改变包装器的成员关系或项目数据格式。
- 不重做全屏 Lookbook 的版式与页数据模型。
- 不改变用户自定义过的文本节点标题。

## 输入 / 输出

- 输入：现有 Flow 节点、`lookbook-membership` 与 `screenplay-page` 关系。
- 输出：单中心包装器端口、无回环成员边、自包含的 3D Lookbook 外观、稳定的包装器开合状态。

## 验收准则

1. Lookbook 类型端口与 `multi` 端口共用中心锚点，画布只显示一个输入锚点。
2. Lookbook/Manus 包装关系使用不会越过端点包围盒的单调曲线路径。
3. Lookbook 在普通、选中状态均无 `.node-card-shell` 背景、圆角和裁切。
4. 页芯无 `repeating-linear-gradient` 横纹，使用实体纸张与少量叠页。
5. 600ms 内重复触发同一包装器开合只接受第一次。
6. 默认标题“Markdown 文本”兼容显示为“文本”，用户自定义标题不受影响。
7. 全屏册子与画布册子内部没有装饰性光影渐变，图片/视频内容不带悬浮投影。

## 约束与风险

- 仅动画 `transform` 与 `opacity`。
- 保留 typed handle 作为隐藏兼容锚点，避免破坏旧连接数据。
- 自定义成员边只作用于包装关系，不影响普通 Flow 连线。
