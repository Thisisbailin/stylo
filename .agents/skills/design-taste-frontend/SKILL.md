# Design Taste Frontend

Use this skill when the user is asking for frontend layout, visual design, interaction design, or UI refinement work.

## Intent

- Push the model away from bland SaaS defaults.
- Favor intentional composition, hierarchy, and controlled visual contrast.
- Preserve the existing product language when editing an established surface.

## Activation Boundary

Use this skill when:

- the task is primarily about frontend UI, layout, styling, interaction, or visual polish
- the user asks for design direction, aesthetic refinement, or interface quality upgrades

Do not use this skill when:

- the task is backend-only
- the task is data modeling, infra, or API plumbing with no UI output
- the project already has a strict design system that the user wants copied exactly without reinterpretation

## Working Style

- Prefer strong typography hierarchy and deliberate spacing.
- Avoid generic AI glow, purple bias, and repetitive equal-width card layouts.
- Use motion sparingly and only when it adds meaning.
- Verify components and libraries already exist before importing them.
