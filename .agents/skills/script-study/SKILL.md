# 剧本学习 Skill

在用户需要你深入理解剧本、角色关系、情节推进、冲突结构、主题线索或叙事问题时使用本 skill。

## 适用场景

- 用户要求你先学习、消化、拆解或总结某一集剧本
- 用户要你判断角色动机、关系变化、情节逻辑、节奏问题
- 用户希望你在后续分镜、角色设计或 workflow 生成前，先建立可靠的剧情理解

## 工作方法

1. 先用项目读取工具定位相关剧本与理解资产，不要直接凭空概括。
2. 优先建立以下结构化认知：
   - 主线冲突是什么
   - 每个关键角色在本段剧情里的目标、阻碍、转折是什么
   - 场景顺序如何推动信息披露、关系变化与情绪升级
   - 是否存在逻辑断裂、情绪跳步、信息缺失或重复
3. 如果材料不足，明确指出缺的集数、场景、角色资料或项目摘要。
4. 输出结论时，优先给出“剧情理解”和“可用于后续工作的事实”，而不是泛泛文学评论。

## 推荐输出框架

- 剧情目标与主冲突
- 关键角色状态变化
- 场景推进链
- 重要视觉/表演信息
- 后续分镜或美术设计需要继承的约束

## 约束

- 不要把未经读取验证的情节当作事实。
- 不要跳过查阅，直接生成“看似合理”的剧情理解。
- 不要过早进入分镜或美术方案，除非用户已经明确要求。
